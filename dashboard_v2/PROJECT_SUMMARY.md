# Dashboard V2 重构项目总结

## 📋 项目概述
将 OpenClaw Health Monitor 的 Dashboard 从复杂的多页面结构重构为简洁的3视图架构。

**重构前**：4个Tab × 10个页面 × 50+卡片 = 信息过载
**重构后**：3个视图（总览/详情/管理）= 一目了然

## 🎯 核心功能实现

### 1. 健康评分系统（health_score.py）

**算法设计**：
```python
基础分：100分
扣分项：
- Gateway不健康：-20分
- 阻塞任务：-10分/个（上限30分）
- 内存压力>80%：-10分
- CPU压力>90%：-10分
- 错误分类：-5分/类（上限20分）
- 学习滞后：-5分

状态分级：
- 90-100：优秀 ✅
- 70-89：良好 🟡
- 50-69：警告 🟠
- 0-49：严重 🔴
```

**下一步行动建议**：
- 立即处理（<50分）：Gateway不健康
- 今天处理（50-70分）：阻塞任务/资源压力
- 本周关注（70-90分）：学习滞后
- 保持现状（>90分）：系统正常

### 2. 数据分级刷新（data_collector.py）

| 数据类型 | 刷新频率 | 缓存策略 |
|---------|---------|---------|
| 健康评分 | 5秒 | 自动缓存5秒 |
| 系统指标 | 5秒 | 自动缓存5秒 |
| 事件时间线 | 5秒 | 自动缓存5秒 |
| 环境状态 | 10秒 | 自动缓存10秒 |
| 代理活动 | 30秒 | 自动缓存30秒 |
| 学习数据 | 60秒 | 自动缓存60秒 |
| 任务列表 | 手动 | 点击刷新 |

### 3. 三视图架构

#### 视图1：总览（Overview）
```
Hero区域（50%视觉权重）
├── 健康评分大圆环（0-100分）
├── 状态指示器
├── 下一步行动卡片
└── 环境切换按钮

KPI条（3个核心指标）
├── CPU使用率 + 进度条
├── 内存使用 + 进度条
└── 会话数 + Gateway状态

事件时间线（可折叠）
└── 最近20条事件记录
```

**特色功能**：
- 设置弹窗：刷新间隔/桌面通知/显示选项
- 自动保存到localStorage
- 实时刷新配置

#### 视图2：详情（Explore）
```
左侧导航
├── 环境
├── 任务
├── 代理
└── 学习

右侧内容区
├── 环境详情
│   ├── Primary 单环境卡片
│   ├── 代码路径/状态路径/Git/PID/Token
│   └── 环境对比表格
├── 任务链路
│   ├── 过滤器（全部/运行中/阻塞/已完成）
│   └── 任务列表
├── 代理活动
│   └── 代理卡片网格
└── 学习监督
    ├── 统计数据（学习项/反思/已晋升）
    └── 学习项列表
```

**特色功能**：
- 一键复制代码路径（悬停显示复制按钮）
- 复制成功提示动画
- 单环境运行态详情视图

#### 视图3：管理（Manage）
```
快照管理
├── 创建快照（输入标签）
└── 现有快照列表（恢复/删除）

运行控制
├── 当前环境显示
├── 重启 Primary
└── 控制台入口

配置管理
├── 自动刷新间隔
├── 钉钉Webhook
├── 飞书Webhook
├── 通知开关
└── 日志级别
```

**高危操作保护**：
- 二次确认弹窗
- 操作说明提示
- 确认后立即执行

### 4. 高危操作设计

**运行控制**：
1. 点击重启按钮 → 显示确认弹窗
2. 弹窗显示操作说明
3. 点击"确认执行" → 执行重启
4. 成功后刷新页面

### 5. API设计（RESTful）

```
GET  /api/v2/status              # 系统状态检查
GET  /api/v2/health/score        # 健康评分
GET  /api/v2/health/next-action  # 下一步行动
GET  /api/v2/metrics             # 系统指标
GET  /api/v2/environments        # 环境列表
POST /api/v2/environments/restart # 重启当前 Primary
GET  /api/v2/events              # 事件时间线
GET  /api/v2/tasks               # 任务列表
GET  /api/v2/tasks/blocked       # 阻塞任务
GET  /api/v2/agents              # 代理列表
GET  /api/v2/learnings           # 学习数据
```
